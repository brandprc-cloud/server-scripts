Партнёры и подрядчики хранятся в `projects/planner/партнёры/` — это эталонная структура.

```
projects/planner/
├── контакты/
│   └── имя-фамилия.md           — контактные данные + ссылка на партнёра
└── партнёры/
    ├── имя-компания.md          — ГЛАВНЫЙ ФАЙЛ (роль, проект, договор, статус, история, следующий шаг)
    └── имя/                     — ПАПКА (только если есть документы)
        ├── договор-название.md  — текст договора + ключевые пункты для претензий
        ├── письмо-ДД.ММ.ГГГГ.md — важные письма
        └── скрипт-возражений.md — сценарии переговоров
```

**Правила именования:**
- Главный файл: `имя-компания.md` строчными через дефис
- Папка с документами: только если есть договор или переписка
- Один файл на человека, даже если работает на несколько проектов

**Связь с задачами:** в `planner/задачи/ПРОЕКТ.md` — строка `Подрядчики: → партнёры/имя.md`

**Why:** Единая точка входа для всех партнёров и подрядчиков. Структура согласована и проверена Андреем 05.06.2026.

**How to apply:** При создании нового партнёра/подрядчика — всегда в `planner/партнёры/`. Папку `projects/contractors/` не использовать — она создана по ошибке в одной из сессий.
